# ImageToWebPConverter 使用指南

此手冊提供給不熟悉程式的同事，協助在 Windows 上使用全新的圖形介面完成批次圖片轉換為 WebP。

## 1. 啟動應用程式
1. 以 Visual Studio 或在資料夾中雙擊 `ImageToWebPConverter.UI.exe`（位於 `ImageToWebPConverter.UI\bin\Debug\net6.0-windows` 內）。
2. 開啟後會看到主要畫面包含來源/輸出設定、轉換選項、進度列表與統計資訊。

## 2. 基本流程
1. **來源資料夾**：點選「瀏覽…」挑選欲轉換的資料夾。支援子資料夾。
2. **輸出資料夾**：預設與來源相同，可重新選擇其他位置。
3. **轉換設定**：
   - 品質：使用滑桿 (1-100)，預設 85。
   - 最大寬/高：填寫像素數；留空表示不縮放。
   - 勾選「包含子資料夾」「覆寫已存在檔案」依需求調整。
4. 按「開始轉換」後，進度清單會顯示每張圖片狀態，可隨時按「取消」停止。
5. 下方統計區會顯示成功/略過/失敗數與耗時。

## 3. 支援的檔案格式
- 來源：PNG、JPG、JPEG、BMP、TIFF
- 產出：WebP，並自動複製來源子資料夾結構

## 4. 常見問題 (FAQ)
- **按鈕呈灰色無法開始？**  
  請確認來源資料夾存在且輸出路徑不為空。選擇來源後會自動帶入輸出。
- **想縮放圖片但不確定尺寸**  
  可只填寫最大寬度或最大高度，系統會依比例調整另一側，確保畫面不變形。
- **需要保留原始檔案**  
  將輸出設定為其他資料夾或取消勾選「覆寫已存在檔案」即可。
- **程式顯示錯誤訊息**  
  進度清單會顯示失敗原因（例如權限不足或檔案被占用），請修正後重新執行；僅會重新處理失敗或被略過的檔案。

## 5. 指令列模式（選用）
若仍需批次腳本，可執行原本的 `ImageToWebPConverter.exe`，依序輸入來源、輸出、品質與其他提示即可，內部同樣使用新的核心服務，結果與 UI 一致。
